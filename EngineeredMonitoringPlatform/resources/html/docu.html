Introduction

The Page Analyzer is a customized, YSlow setup based on YSlow, which provides various additional functionality.

How it works:

  • It will analyze all the data present in the HAR-File and will present them in one statistic.
  • You can analyze a single page or multiple pages. If you analyze multiple pages it will aggregate all the pages in the result.
  • The rules and the analysis is highly customized and is not comparable to other YSlow tools.
  • The rules rather blame to much, therefore it might give false positives.
  • The results always have to be evaluated through some human mind(or other species with similar intelligence). As anyway each action you do in our application needs time, a rule might blame it even if it is the best way to do it in your page architecture.

Step 1: Create a HAR File

Create a .har-File(har=HTTP Archive) using Chrome, Firefox, or Fiddler.

  • Page Analyzer: Go to the tab "Analyze URL" and enter the URL that should be analyzed. This works only when the application server has internet access, and the page is accessible without a login.
  • Chrome: Open the developer tools, capture network traffic, right click and save as .har-File.
  • Firefox: Open the developer tools, capture network traffic, right click and save as .har-File.
  • Internet Explorer: Use Fiddler. IMPORTANT: The rule "Compress with GZIP" will show wrong results when using Internet Explorer, as it doesn't save in the .har-File if a response is gzipped.
  • Fiddler: Capture network traffic and go to "File >> Export Sessions >> All Sessions" and choose "HTTP Archive v1.2" as export format.

Step 2: Upload HAR File

After creating a .har-File upload it on the page "HAR Upload". The summary will be displayed, please be aware that in the menu bar additional items where added where you can get other views as well. When the checkbox "Save Results" is enabled, the result and the .har.File will be saved to the database and will be accessible through "Menu >> History".

Step 3: Analysis

Please consider the following when analyzing your websites performance with the Page Analyzer:

  • Arbitrary Weighting: The weights of the rules and their contribution to the overall score were in most cases defined arbitrary. It depends on your websites architecture what will have a bigger impact and what not.
  • 100% not achievable: When your website reaches a certain size, a score of 100% can get unreachable. Most pages will end up with a score between 60-90%.
  • Use Scores as hints: Don't threat findings always as bugs, rather consider them as opportunities to improve your websites performance. Performance is always a trade for functionality and other things you want to have. Some of the bad grades might be there on purpose.

Result View

The following views are available from the menu when viewing a result:

  • Summary: Shows a summary of all requests on the page, the component statistics and grades. Click on a grade to open the panel and view the details.
  • Grade >> Panels: Show the rule grades as panels.
  • Grade >> Table: Show the rule grades in a table.
  • Grade >> Plain Text: Show the rule grades as plain text.
  • Grade >> JIRA Ticket: Show the rule grades formatted in JIRA syntax that can be copy pasted into JIRA.
  • Grade >> CSV: Show the rule grades in CSV-Format(for export).
  • Grade >> JSON: Show the rule grades in JSON-Format(for export).
  • Table >> Statistics by Type: Show statistics by request type(assuming empty cache).
  • Table >> Statistics by Type with primed Cache: Show statistics by request type(assuming reading cache).
  • Table >> Components: Show all components in a table.
  • Chart >> Gantt Chart: Show a Gantt Chart for the analyzed page. Click on the magnifying icon to get more details for the request.

Gantt Chart View

The Gantt Chart View will provide you the following:

  • Timings Breakdown: A table with all requests and their timings in a gantt chart style.
  • Details: Clicking on the magnifying icon will give you a lot of details for the provided request.
  • Analyze Cookies: Select a cookie from the dropdown to get all the values for this specific cookie for all the requests where it appears.
  • Analyze Headers: Select an HTTP header from the dropdown to get all the values for this specific header for all the requests where it appears.

History View

The history view will provide you the following:

  • Table: A table with all your results.
  • Table Filter: Enter a keyword in the filter box to search the table.
  • Eye Icon: Open the result view.
  • Sliders Icon: Open the Gantt Chart.
  • Link Icon: Open the link of the analyzed website in a new tab.
  • Trash Icon: Delete the result.
  • Compare Button: Select two or more results and click the Compare button below the table to get a compare view.
  • Delete Button: Select one or more results and click the Delete button for deleting the results.

Comparison View

The comparison view will show you key metrics for each result selected on the history view.
