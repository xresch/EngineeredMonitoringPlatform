InfluxDB Widgets Setup

Before the InfluxDB widgets can be used, at least one InfluxDB environment has to be defined by a user with admin permissions. The environments can be added in the Context Settings found under "Admin >> Context Settings".

The Query Placeholders

In order to get the time picker working properly with the widgets, placeholders for time and interval have to be added to the queries. The following placeholders are available:

  • [earliest]: This placeholder will be replaced by the earliest time choosen with the time picker in nanoseconds.
  • [latest]: This placeholder will be replaced by the latest time chosen with the time picker in nanoseconds.
  • [interval]: This placeholder will be replaced by an interval string(1m, 5m, 1h, 24h etc...). The interval is based on the span selected timeframe, to provide a good granularity while keeping performance in mind.

When a InfluxQL widget is created, it will contain the following example query, that shows how the placeholders can be used:

SELECT mean(*)
FROM runtime 
WHERE time >= [earliest] and time < [latest] group by time([interval]);

Widgets

InfluxQL Threshold

The InfluxQL Threshold widget is used to execute a InfluxQL query and display its results as tiles, panels or as a table. The widget fetches the data of the latest time picked with the time frame picker. You can define thresholds for the values you retrieved to indicate a status with various colors.

Compatibility Notice: This widget is using the API version v1 used in InfluxDB 1.x. As of the official documentation, this should also work with InfluxDB 2.x if the bucket is mapped to a database.

InfluxQL Chart

The InfluxQL Chart widget is used to execute one or multiple queries and display the results as a chart. The widget fetches the data for the time chosen with the time frame picker. The legend in the chart will use the metric labels for the title of the series.

Compatibility Notice: This widget is using the API version v1 used in InfluxDB 1.x. As of the official documentation, this should also work with InfluxDB 2.x if the bucket is mapped to a database.
