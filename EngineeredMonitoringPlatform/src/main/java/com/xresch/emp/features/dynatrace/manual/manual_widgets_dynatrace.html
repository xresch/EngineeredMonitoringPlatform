Dynatrace Widgets Setup

Before the Dynatrace widgets can be used, at least one Dynatrace environment has to be defined by a user with admin permissions. The environments can be added in the Context Settings found under "Admin >> Context Settings". The settings will need the API URL and an API token from the Dynatrace instance.

API Limitations

The API has the following known limitations;

  • SaaS Requests: Dynatrace SaaS solution has a limitation of 50 request/min. Extensive use of Dynatrace widgets with can easily reach this limit and will result in error messages. This is not the case with on premis solutions.
  • Timeframes: Various API calls have limitations on how much data they will return. In case such a limit is reached, a warning message is displayed for each widget running into this issue.

Widgets

Host Details

With the Host details widget you can display details about a host, like the operating system type or the number of CPU cores.

Host Events

This widget will list the events which occurred in the selected timeframe. The maximum timeframe is 2 years.

Host Logs

Will display logs extracted for the selected host and operating system log file. You can filter the logs with the same query syntax used on the Dynatrace UI.
