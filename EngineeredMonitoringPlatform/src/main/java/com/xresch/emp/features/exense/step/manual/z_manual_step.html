Setup

Before the Step features can be used, at least one Exense STEP connection has to be defined by a user with admin permissions. The environments can be added in the Context Settings found under "Admin >> Context Settings". Hover the info icon next to the settings fields to get more info about what you need to fill in. When saving the STEP context setting, all the scheduler will be reloaded, what might take several minutes depending on the number of schedulers you have.

Widgets

Several Widgets are included in the Exense Step feature. Please find their documentation in this manual under section Dashboard >> Widgets >> Exense Step

Widget Status Colors & Thresholds

The status colors displayed are defined as follows:

  • Step Result:  The result of the STEP execution, like RUNNING,PASSED, FAILED etc...
  • Custom Performance Thresholds(optional):  Most STEP widgets provide threshold fields. If the execution has status PASSED, and at least one threshold is defined, thresholds will be evaluated. Define a time in milliseconds that will evaluate the duration of the scheduler execution.

The colors are as follow:

  •  
    Green:  PASSED / Custom Threshold
  •  
    Lime Green:  Custom Threshold
  •  
    Yellow:  Custom Threshold
  •  
    Orange:  VETOED / SKIPPED / INTERRUPTED / Custom Threshold
  •  
    Red:  FAILED / Custom Threshold
  •  
    Black:  BLACK
  •  
    Black:  RUNNING

Widget Performance

The performance of the widgets depends on the following factors:

  • STEP:  The capacity and infrastructure available to the STEP instance.
  • Widget:  The widget you are using and what settings you have set.
  • Data & Timeframe:  The amount of data which is loaded. For time-based widgets, this can highly be influenced by the size of the timeframe you select.

API Endpoint

The EMP API provides an endpoint named "ExenseStep" to fetch execution status data from STEP. Please go to the API UI to find the documentation about the available requests and create request examples.
